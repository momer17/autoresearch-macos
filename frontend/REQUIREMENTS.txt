API CONTRACTS
-------------
POST /start — multipart/form-data
  file             File      required — CSV upload
  target_col       string    required
  task_description string    required
  total_iterations number    optional  default 8

  *** DO NOT send metric, task_type, or feature_cols — backend auto-infers all three ***

POST /start — response JSON
  experiment_id    string
  status           "started"
  task_type        string    (binary_classification | multiclass_classification | regression)
  metric           string    (roc_auc | f1 | f1_macro | rmse | ...)

  *** Store task_type and metric from this response — they are NOT in /status ***

GET /status — JSON
  status           "idle" | "setting_up" | "running_baseline" | "running" | "complete" | "error"
  experiment_id    string | null
  baseline         number | null
  best_score       number | null
  current_iteration number
  total_iterations  number
  research_summary  string
  iterations        Iteration[]
  error             string | null

Iteration {
  iteration   number
  score       number | null   (null = crashed run)
  best_score  number
  strategy    string
  kept        boolean
  error       string | null
}

KEY RULES
---------
1. API proxy target in vite.config.ts is the only place to set the GPU IP.
2. No mock data — backend is live.
3. Poll every 3 seconds. Stop on "complete" or "error".
4. metric and task_type come from the /start response, NOT from /status — stored in App state.
5. Metric direction: higher is better for f1/accuracy/roc_auc/r2 — lower for rmse/mae.
6. score in iterations can be null — always null-check before displaying.
7. No UI libraries needed.
8. Input bar is always visible — never hide it except during "starting" stage.
